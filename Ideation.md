# Ideation
이전 과제 내용이 be 레포에 생성되어 있어 추가 생성하였으니, 업로드 일자를 확인하려면 [이 부분](https://github.com/26-ewha-capstone-logue/logue-be/blob/main/Ideation.md)을 클릭해주세요.

## 1. 누구를 위해
데이터 기반 의사결정을 해야 하지만 SQL이나 데이터 구조를 이해하기 어려운 기획자, 마케터, 비개발 직군을 위해.

## 2. 어떤 문제를 해결하려 하는가
비즈니스 데이터를 확인하기 위해 개발자에게 SQL 쿼리를 요청해야 하는 구조로 인해 데이터 접근이 느리고 반복적인 커뮤니케이션 비용이 발생한다.  
또한 간단한 질문에도 분석이 지연되어 빠른 의사결정이 어렵다.

## 3. 어떤 기술을 사용하는가
- LLM 기반 **Text-to-SQL**
- 데이터베이스 **Schema 이해 및 매핑**
- 자연어 질문을 SQL로 변환하는 **Query Generation**
- 결과를 시각적으로 보여주는 **데이터 조회 인터페이스**

## 4. 무엇을 만들려고 하는가
사용자가 자연어로 질문을 입력하면 데이터베이스를 분석하여 적절한 SQL을 생성하고 결과를 반환하는 **데이터 질의 인터페이스**.

예시  
> "최근 한 달간 서울 지역 신규 고객의 평균 결제 금액은?"

## 5. 기대 효과
- 비개발자도 **직접 데이터를 조회**할 수 있음  
- 개발자 의존도를 줄여 **분석 속도 개선**  
- 반복적인 데이터 요청 감소  
- 데이터 기반 의사결정 환경 강화
